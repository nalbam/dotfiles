# CLAUDE.md

이 파일은 이 저장소에서 코드 작업을 할 때 Claude Code (claude.ai/code)에게 지침을 제공합니다.

# 프로젝트 코드 가이드라인

**중요: 항상 `docs/` 디렉토리의 주요 개발 문서를 참조하세요.**

**중요: 새 코드를 작성하기 전에 유사한 기존 코드를 검색하고 일관된 패턴을 유지하세요.**

**중요: 필요한 작업만 수행하세요. 작업이 필요하지 않으면 중단하세요.**

## 마인드셋

- 시니어 엔지니어처럼 생각하세요.
- 성급히 결론내리지 말고, 결정 전에 여러 접근법을 평가하세요.
- 문제 정의 → 작고 안전한 변경 → 리뷰 → 리팩터링 — 이 루프를 반복하세요.
- 커밋과 PR은 작고 집중적으로 유지하세요.

## 코드 변경 전 확인사항

- 호출/참조 경로를 포함하여 관련 파일을 처음부터 끝까지 읽으세요.
- 정의, 참조, 호출 지점, 관련 테스트, 설정을 찾아 읽으세요.
- 파일 전체를 읽지 않았다면 코드를 변경하지 마세요.
- 가정은 명확히 기록하세요.

## 핵심 원칙

- **올바른 문제 해결**: 불필요한 복잡성이나 범위 확대를 피하세요.
- **표준 솔루션 선호**: 커스텀 코드 작성 전에 잘 알려진 라이브러리와 패턴을 사용하세요.
- **가독성 유지**: 명확한 이름, 논리적 구조, 깊은 중첩 회피.
- **명시적 오류 처리**: 구체적 예외 처리, 의미 있는 메시지로 빠르게 실패.
- **보안 설계**: 입력 검증, 최소 권한 적용, 비밀값 노출 금지.
- **얕은 의존성 유지**: 긴밀한 결합 최소화, 명확한 경계 유지.
- **근본 원인 해결**: 증상이 아닌 근본 문제를 수정하세요.

## 파일 크기 가이드라인

- 소스 코드 파일은 700줄 미만으로 유지하세요.
- 500줄 이상이면 리팩토링을 고려하세요.
- 큰 컴포넌트는 작고 집중된 모듈로 분할하세요.

## 테스팅 전략

- 중요한 로직과 사용자 플로우에 대해 테스트를 작성하세요.
- 버그 수정 시 회귀 테스트를 반드시 포함하세요.
- 테스트는 빠르고, 격리되고, 결정적이어야 합니다.

## 보안 규칙

- 코드, 로그, 커밋에 비밀값을 절대 노출하지 마세요.
- 모든 입력을 검증, 정제, 인코딩하세요.
- 최소 권한 원칙을 적용하세요.
- 민감한 데이터를 로깅하지 마세요.

## 피해야 할 안티패턴

- 전체 문맥을 읽지 않고 코드 수정.
- 실패, 경고, 엣지 케이스 무시.
- 섣부른 최적화나 추상화.
- 광범위한 예외 핸들러 사용.
